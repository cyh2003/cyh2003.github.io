Introduction to Computer System

基本数据类型

  1. 16进制开头要写0x
  2. 常数默认是带符号数，表达式中若混有无符号数和带符号数，则带符号数默认转换为无符号数（尤其在比较运算中需要注意）
  3. 注意带符号数的上限的绝对值比下限的绝对值少1
  4. 无符号数右移为逻辑右移，带符号数右移为算数右移，均向负无穷方向舍入，如要使带符号数在为负数时向0舍入，需使用(x+(1<<k)-1)>>k
  5. 浮点数包括规格化浮点数、非规格化浮点数、特殊值，对于规格化浮点数，（此时不包括全1或全0值），，尾数隐含第一位的1，对于非规格化浮点数，为全0，（即实际的指数与规格化浮点数时相等），尾数不再隐含1，对于特殊值，为全1，当为全0时表示，当不为全0时表示NaN
  6. 由于浮点数的编码特点，浮点数既有又有
  7. 浮点数舍入采用Nearest Even规则，从而避免统计误差（如四舍五入会使得平均值偏高）
  8. 浮点数舍入可能导致尾数的溢出，此时还需要进一步调整指数位
  9. 浮点数转换为int时将尾数部分截断，如果溢出或浮点数是NaN，则通常转换为Tmin或Tmax，int转换为double可以精确转换，int转换为float不会溢出，但可能被舍入

指令集

  1. 汇编指令操作的数据类型的长度按b、w、l、q递增，分别为1、2、4、8字节
  2. 简单寻址的形式为(R)、D(R)，对寄存器无特殊要求，变址的通用形式为D(Rb,Ri,S)，有(Rb,Ri)、D(Rb,Ri)、(Rb,Ri,S)、(,Ri,S)、D(,Ri,S)等变型，其中D为常数，Rb为16个通用寄存器之一，Ri不能为rsp，S为1、2、4、8中的一个值
  3. movl指令的目的操作数若为寄存器低32位，则会将目的寄存器的高32位置为0（实际上x86-64中多数目的操作数位寄存器低32位的指令都有这种特性）
  4. movs系列表示有符号扩展传送，movz系列表示无符号扩展传送
  5. 算逻指令可以以内存数据为操作数
  6. 对于mov类指令，对标签加$将得到标签所在的地址，不加$将得到标签后数据元素对应的值

控制流

  1. 注意CF、SF、ZF、OF的含义，以及解释条件跳转与条件码的关系
  2. leaq指令不设置条件码
  3. cmp指令的语义与操作数的左右顺序相反，可以认为是不对目的操作数修改的sub指令
  4. cmp指令也有目的操作数的概念，其不能为立即数，在AT&T标准下，即第一个操作数可以为立即数，第二个操作数不能为立即数
  5. test指令计算操作数的与但不改变操作数，与cmp指令类似，其目的操作数不能为立即数
  6. setX系列指令会读取当前的条件码（或某些条件码的组合），根据判断类型X是否为真，将真值（0或1）存入目的寄存器的低位字节，余下的7个字节不会被修改，此时通常使用movzbl指令进行零扩展
  7. 直接跳转的跳转地址直接编码在目标文件中，在汇编中一般为跳转到某一标号，而间接跳转依靠寄存器中的值及相关地址计算结果作为跳转地址
  8. 条件跳转只能使用直接跳转
  9. 直接跳转之所以可以在编译期解决，是因为其机器码中实际储存的是跳转目标指令与下一条指令的“字节距离”，这在链接中是不会改变的，间接跳转需要在链接期解决

函数调用

  1. 如果函数的参数太多（超过6个），无法全部放入寄存器，那么剩下的参数将在函数调用时反向压栈（即参数在参数列表中越靠后，地址值越大）
  2. 在x86-32下，如果函数返回值为64位，则会将高位保存在edx中，低位保存在eax中进行返回，同理，在x86-64下，如果函数返回值为128位，则会将高位保存在rdx中，低位保存在rax中进行返回
  3. 两个栈帧的划分为，保存的rip寄存器（返回地址）属于上一个栈帧，保存的rbp寄存器（可选）属于下一个栈帧
  4. 栈指针rsp是特殊的，除此以外，rbx，rbp，r12，r13，r14，r15是callee save，其他的寄存器均是caller save
  5. rax寄存器一定是caller save，因为callee不可能在退出前恢复rax，而这又是因为callee有自己的返回值占用rax
  6. rip寄存器无法直接赋值，只能通过call、ret、jmp等指令间接赋值
  7. x86-64可以延迟分配，其含义为可以直接访问不低于当前rsp寄存器128字节的栈上空间，这段空间被称为Red zone
  8. x86-32的函数调用与x86-64不同，首先，rbp寄存器固定作为栈帧指针使用（callee save），指向当前栈帧的"底端"，其次，所有参数都通过栈来传递
  9. -fno-omit-frame-pointer参数用来使编译时不忽略栈指针

数组、结构、联合

  1. 数组计算内存地址时注意元素自身的大小（例如int为4字节）
  2. 注意嵌套数组与多层数组内存访问方式的不同
  3. 变长数组通过rsp寄存器根据参数确定变化量来实现
  4. 结构体内存对齐规则共4条，首先，第一个成员在结构体变量偏移量为0的地址处，其次，其他成员变量要对齐到某个数字（对齐数）的整数倍的地址处，对齐数一般为成员自身表示数据的内存大小（除long double外），再次，结构体总大小为最大对齐数的整数倍，最后，如果嵌套结构体，嵌套的结构体对齐到自己的最大对齐数的整数倍处，结构体的大小为所有对齐数/最大对齐数（包含嵌套结构体的对齐数）的整数倍
  5. 对齐总的原则是让每个基本类型的变量都能保持对齐
  6. long double的对齐要求为16字节，与其自身表示数据的内存大小不相同
  7. x86-64下，若结构体大小不适合64位，结构体作为返回值通过传入一个隐式的指针参数作为函数的第一个参数来实现，结构体作为参数可以通过在父函数的栈帧中拷贝一份作为参数实现

链接

  1. 未初始化全局变量属于COMMON，故未初始化全局变量在链接完成后才会被分配到具体的段（本质上这样做是因为该符号是弱符号，有可能在别的文件也被定义并最终采用），.bss段包括未初始化静态变量，以及初始化为0的全局变量或静态变量
  2. 内联函数和静态函数在编译期定位
  3. 静态变量与全局非静态变量一样，在链接期进行定位
  4. gcc默认自动链接标准库
  5. 使用ar命令可以将多个.o文件构建成一个.a静态库文件
  6. 在动态链接库中的静态变量和非静态全局变量的访问方式不同，非静态全局变量需要通过.got间接取址访问

虚存

  1. PTE中含有标志位，用于标识权限，起到保护作用，当发生违法访问时，向进程发出SIGSEGV信号
  2. TLBI为TLB index，对应虚拟地址三部分的中间部分，TLBT为TLB tag，对应虚拟地址三部分的开头部分
  3. 注意虚拟地址和物理地址的取值范围，不要被16进制表示所干扰
  4. 内存映射（memory mapping）指初始化虚存空间并将其与硬盘对象关联
  5. regular file对应可执行文件中的初始化全局数据段和代码段，anonymous file对应未进行写操作的栈和堆，同时采用demand-zero page使得分配的物理页面内容为0，swap file对应所有写入过的页面
  6. COW在一个进程与另一个进程共享一个私有页，并且一个进程试图对其进行写操作时发生（fork可以导致这一情况），具体过程为写操作指令触发异常（protection fault，由于之前双方进程的flag都已经被置为read-only），handler为其复制一个新的页面用于映射，然后指令重新执行
  7. demand paging使得并不是所有虚存在一开始全部载入内存，而是在有访问请求时才装入内存

动态内存分配

  1. 内部碎片化源于块大小大于有效载荷，外部碎片化源于总空闲内存不小于申请的块大小，但任意单独的块大小都小于申请块
  2. 之所以可以用块中header的低位存储flag，是因为内存对齐，使得header中size的低位一定是0，可以用来存放其他信息
  3. 在implicit list中，使用boundary tag检测前一个块的状态会增加内部碎片化，通过在header中增设一个标记前一个块状态的flag
  4. 可以通过延迟空闲块的合并来提升性能

异常控制流

  1. 通过画新进程产生的流程图帮助判断进程间的关系
  2. 子进程在刚诞生时拥有与父进程相同的打开文件列表
  3. atexit函数可将参数中的函数注册（必须参数为空，返回为空）， 当程序正常终止时，自动调用被注册的函数
  4. wait函数和waitpid函数都会阻塞的等待子进程的结束
  5. WIFEXITED宏用于判断子进程是否正常终止，WEXITSTATUS宏用于得到子进程退出值
  6. execve函数用来加载和运行可执行程序，除非错误否则不返回，覆盖原来的代码段、数据段和栈，但保留进程号，打开文件列表和信号上下文

信号处理

  1. 进程间通过kill函数来传递信号（也可以一次性向一个进程组中的所有进程发送信号）
  2. 被目标进程阻塞的信号可以被delivered从而进入pending状态，但不能被recieved
  3. 信号处理函数（handler）有独立的上下文，其执行不会受主程序状态（例如是否阻塞）的影响
  4. 当进程恢复执行时，若存在可以接受的信号，则从最低位可接受信号开始依次执行
  5. SIG_IGN作为handler传入表示忽略信号，SIG_DFL作为handler传入表示恢复默认处理
  6. 当多个同类型信号在短时间内到达，可能会造成信号丢失，可能会导致一些操作的遗漏，往往可以通过增加while循环来检查
  7. alarm()函数负责发送SIGALRM信号，alarm(t)中t作为参数表示延迟t秒发出信号
  8. 一个函数是异步信号安全的当且仅当仅有局部非静态变量或不可被信号打断
  9. 使用setjmp函数和longjmp函数可以实现非本地跳转，但只能跳转到已被调用且未结束调用的函数，使用sigsetjmp函数和siglongjmp函数可以实现由信号触发的非本地跳转（原理可参考作业中x86-32下的GET和SET函数）
  10. 隐式阻塞机制，即内核默认阻塞当前正在处理信号类型的待处理信号，使得单个进程的单个信号后者不会打断前者

文件I/O

  1. 成功的关闭并不确保数据已经成功地保存到磁盘，因为内核会延迟写操作，如果需要确保，需要在关闭前使用fsync函数
  2. Unix I/O会有不足值的问题，需要进行处理
  3. fcntl函数可以用来对打开的文件描述符进行读取或设置各种属性
  4. 调用dup和dup2后，复制出来的文件描述符与原来的文件描述符指向同一个文件表项，共享文件偏移量，实质上仅仅是对原来的文件表项增加引用计数
  5. 在打开文件时使用 O_TRUNC会覆盖原来的所有内容
  6. Unix I/O在读入时不会追加终止字符'\0'
  7. 当Unix I/O以O_RDWR（读写）打开文件时，读和写操作共享一个文件偏移量
  8. 当Unix I/O以O_APPEND（追加）打开文件时，进行写操作时每次都会检查当前文件真正的末尾并追加，进行读操作则从头开始读取

多线程

  1. 线程有独立的CPU上下文（通用寄存器，条件码，栈指针，程序计数器）和栈空间，其他的均共享
  2. 多线程可以在多核处理器下实现真正的并行
  3. pthread_join函数用于控制线程阻塞地等待指定线程的结束，获取其状态并清除其占用的内存（类似进程层面的wait函数）
  4. pthread_detach函数在线程的例程中使用，用于使pthread_detach传入参数中的线程在结束时自动释放，而不需要其他线程的回收，与pthread_join函数互斥
  5. pthread_exit用于当前线程退出，而pthread_cancel是当前线程取消另一线程的执行
  6. 一个变量被共享，当且仅当有多个线程引用该变量的（至少）一个实例

正向编程

  1. 对于汇编正向编程，可执行程序通过命令行启动时，命令行参数argv[0]、argv[1]等从低地址向高地址依次被保存在栈中
  2. 汇编代码调用C函数时有时需要使用动态链接器链接标准库，C代码作为主函数调用汇编函数时正常链接即可
