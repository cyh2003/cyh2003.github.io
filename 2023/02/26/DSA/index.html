    PA

    tips

    1. 对于Windows，由于没有现成的重定向输入（cat命令+管道会产生一些问题），最好使用freopen进行重定向输入
    2. 在一些情况下，使用C++将字符输出到文件中时，默认以UTF-16 LE编码，可能会导致一些问题，修改为UTF-8可以解决（Lab 1 Zuma）
    3. OJ获取数据范围的办法为，读入数据范围并使用exit函数将数据范围的值作为错误码，然后观察反馈，由于一次只能观察1字节，故需要不断除以256用于看到更高位的值，另外需要注意，如果出现WA，说明exit(0)，一般来说表示打印的字节为数据的高位0或低位0
    4. OJ获取发生错误的代码位置为，使用exit函数在源代码的执行流中进行二分查找
    5. 一些情况下开启-O2优化可能导致程序运行结果的不同（PA 4-1 Circuit中导致RE，最后通过递归改迭代解决）
    6. 不进行delete操作，将内存清理交给操作系统，有时可以显著降低运行时间（但可能同时增加虚拟内存的消耗，可能导致MLE）
    7. 对于LAB 3 BBST，课程示例代码多有错误，尽量不要参考，可选择参考https://algs4.cs.princeton.edu/中的代码，其中AVL的实现没有问题，Splay的实现需要将remove公有方法中第二处对splay私有方法的调用之前加上root = ，以保证根节点的正确性

    Theory

    Basic

    1. 迭代对数定义为$\log^n=\left{\begin{align}0&&if\ n\le1;\1+\log^(\log n)&&if\ n\gt1.\end{align}\right.$，增长速度远慢于对数
    2. 若某一问题存在算法，则所有算法中复杂度最低（有可能是最好情况，最坏情况，平均情况）的称为问题的难度
    3. 主定理（Master Theorem）表明，设，满足，，情形1，若存在常数，有，则，情形2，若存在常数，有，则，情形3，若存在常数，有，且存在常数和充分大的，满足，则
    4. 递归函数调用次数计算时，注意不要忽略根节点的第一次调用
    5. Karatsuba算法可以将长整数相乘的时间复杂度由降为
    6. 由于总和最大区段的最大性和最短性，故其要么和最短非正后缀无交，要么是最短非正后缀的真后缀，故可以利用减治的思想，通过一次逆向扫描得到答案，时间复杂度为
    7. 双亲与孩子为互逆关系，祖先与后代为互逆关系
    8. 各种有序（升序）结构的search操作一般约定为返回最大的不超过传入参数大小的元素的最大（若有多个符合条件的元素）秩
    9. 树中节点的度为孩子的数量，有序树为规定了孩子先后次序的树
    10. 注意与树相关的算法往往需要注意维护根节点指针的值
    11. 在无特殊说明的情况下，一般讨论简单图
    12. 图中的路径长度一般为边的个数而非点的个数
    13. 图算法中“按合成数字典序”的“合成数”即指构成的元组
    14. 图中点的状态在遍历中一般分为UNDISCOVERED、DISCOVERED、VISITED三种，分别对应尚未访问到、已经访问到、已经访问并处理完毕
    15. 图中边的状态在遍历中一般分为TREE、FORWARD、BACKEND、CROSS四种，分别对应遍历树（森立）中的边、遍历树祖先节点连向真后代节点的边（不包括树边）、遍历树后代节点连向真祖先节点的边、除以上三者外的边（跨边）

    Vector

    1. 分三种情况（可提前命中）的二分查找时间复杂度满足递归式，由此可以解得常系数最小时取
    2. Hopcroft校验环在删除时，实质操作为将最后一项校验环挪到被删除校验环的位置处，并前移尾指针
    3. 游标本质上是在两个向量（分别为数据向量和下一项的下标向量）内储存一个数据链表和一个空闲链表，插入元素时数据链表头插，空闲链表头删，删除元素时数据链表删除对应位置，空闲链表头插

    Stack & Queue

    1. 栈混洗数即卡特兰数，通项公式为，可由"不合法"的push+pop序列与push比pop数量多2的全排列一一对应这一事实得出（对应方法为"不合法"序列一定有第一次pop比push多1的位置，将这一位置之前push和pop翻转，则push比pop数量多2，而反之任一这样的全排列均可转换回去）
    2. 栈混洗数递推公式为，可考虑栈顶元素入栈的位次，由于栈顶元素入栈时中转栈必为空，之前入栈和之后入栈的元素次序彼此独立，可得递推公式，用这种方法和数学归纳法还可证明栈混洗一定不含禁形
    3. 证明“不含禁形则一定是栈混洗”这一命题，可以通过用栈混洗数递推公式的类似方法和数学归纳法得出
    4. 证明“915”式禁形和“615”式禁形等价性，需要对三个元素的极差做数学归纳
    5. 通过模拟栈混洗的过程，可以得到的栈混洗鉴别算法
    6. 将中缀表达式手动转换成RPN需要先补全括号，然后以运算符代替右括号，清除左括号
    7. Steap在不断的插入/删除操作中，通过插入/删除最值（不需要维护后缀最值的计数），用时间复杂度，维护一个后缀最值Stack
    8. Queap在不断的插入/删除操作中，通过插入/删除当前元素（由于Queue的性质，需要维护后缀最值的计数，计数的总和等于当前元素数量），用时间复杂度（采用计数才能达到此复杂度），维护一个后缀最值Queue（实际上就是维护一个带计数的单调队列）
    9. 直方图最大矩形的单次扫描算法在每轮迭代中，通过维护一个单调栈，计算以当前位置为矩形最右侧边界的最大矩形面积

    Tree

    1. 多叉树可以通过兄弟节点间的关系来优化算法
    2. 注意祖先/后代（含自身）和真祖先/后代（不含自身）的区别
    3. 树的半线性体现为，在同一深度中任一节点的祖先唯一但后代未必唯一
    4. 空树高度-1，单节点树的高度和深度均为0

    BT

    1. 有根有序树等价于二叉树，可以通过将多叉树中每个节点的长子作为左孩子，第一个右兄弟作为右孩子来转化为二叉树
    2. 二叉树的前序遍历和中序遍历有多种非递归的实现方式，其中中序遍历可不借助栈来实现（利用中序遍历的本质是不断迭代后继）
    3. 二叉树的后序遍历中栈的特点是，一个节点若为左孩子，则栈中的上一个节点就是右兄弟，一个节点若为右孩子，则栈中的上一个节点就是父节点
    4. 完全二叉树由于其特殊结构，叶节点数量与内部节点数量相等或多1，因此层次遍历中辅助队列的规模单峰且对称
    5. 给出BT的前序遍历或后序遍历，再加上中序遍历可以还原出BT，只给出BT的前序遍历和后序遍历无法还原出BT，但若BT是真BT，或者遍历序列是增强序列（即对所有只有一个孩子的节点补充一个空子节点，对所有叶节点补充两个空子节点），则可以还原出BT
    6. 当各个字符出现的频率相同时，Huffman树一定可以构造为真完全树，因为叶节点一定位于倒数两层内，否则可以一直交换节点转变为真完全树
    7. 对字符集大小做归纳，可以得出Huffman树一定是最优编码树之一

    BST

    1. 对于允许重复关键码的搜索树，注意相应的searchAll接口和searchFirst接口的实现（见习题解析7-10、7-16、8-3）
    2. _hot节点是命中节点（可为空）的父节点
    3. 对于最朴素的实现，搜索算法命中的节点和作为搜索目标的值之间的大小关系不能确定
    4. 删除节点要分所删节点有0、1、2颗子树进行不同的操作
    5. 按不同的插入顺序等概率分布时，平均树高为，按不同的拓扑结构等概率分布时，平均树高为
    6. zig(v)为以v为轴向右旋转，zag(v)为以v为轴向左旋转，二者为互逆操作
    7. 任何等价的二叉搜索树都可以通过次旋转相互转化（大体思路为在最左侧分支自下而上不断旋转，得到一条单链，详见习题解析7-15）

    AVL Tree

    1. 节点的平衡因子可定义为左子树高度减去右子树高度，AVL树始终满足每一个节点的平衡因子绝对值不超过1，由此通过对某一树高下AVL树的最少节点数公式（）的归纳证明，可以保证AVL树的树高一定为
    2. 删除操作需要根据删除节点的孩子情况进行分类处理，在两个孩子均非空时需要与后继节点交换（不一定为叶节点）
    3. 插入操作可能导致插入节点的所有祖先同时不平衡，删除操作可能导致删除节点的任一祖先不平衡，但同时最多有一个，在重平衡时每层至多检查平衡因子并旋转一次，同时更新高度
    4. 在插入和删除时，若"关键三节点"呈"一"字型，则进行单旋，若"关键三节点"呈"之"字型，则进行双旋
    5. 在插入时最多进行一次调整则全树重平衡，在删除时可能需要进行次调整才能重平衡
    6. 对于各种旋转操作，可以全部统一为3+4重构，即找到最低失衡节点g，沿较长分支依次找到g、p、v三节点及它们的四颗子树，按照中序遍历的顺序进行调整
    7. AVL树的优势是保证查找、插入、删除最坏复杂度均为，只进行插入和查找时优于红黑树
    8. AVL树的劣势是需要维护高度和平衡因子，需要为此改造元素结构，需要的旋转操作次数相较于红黑树更多，在频繁插入和删除时性能不如红黑树，单次调整之后全树拓扑结构结构变化量可达

    k-D Tree

    1. k-D树长方形区域（超长方体）查询的时间复杂度可以通过各个边界切割区间次数之和来进行计算（这里的区间是查询过程中遇到的所有区间，包括不是“叶节点”的区间，且递归调用的次数不多于切割区间次数之和），对于一个边界的切割次数，可以通过递归表达式，得，从而在忽略输出各个点信息的情况下，时间复杂度为
    2. 对于k-D树矩阵（张量）查询的时间复杂度最坏情况的构造，可以设k-D树均匀的分割了空间，然后在沿任意一个边缘取一k-1维的区域进行查询，则此时复杂度达到上界（不考虑遍历所有命中的几何节点的时间）
    3. k-D树并不一定要使叶节点只包含一个几何节点（但默认如此），在矩阵查询或者最近邻问题中，可以令一个叶节点包含多个几何节点，在叶节点内采用枚举，可以加快运行速度

    Multi-level Tree

    1. 与k-D树不同，多层树是一维一维依次构建的数据结构，先按x坐标构成第一层BBST，第一层BBST中的每个节点所包含的几何节点再按y坐标组织成第二层BBST
    2. 当空间维数为，多层树空间复杂度（考虑不同区间大小的节点的大小范围、个数和包含集几何点的数量，进行求和），初始化时间复杂度（由几何点构成的最底层BBST均为最后一维有序，需在构造子节点的过程中采用中位数选取算法得出中位数，然后依次比较进行分割，如此才能达到现在的复杂度），单次查询的时间复杂度（不考虑遍历所有命中几何节点的时间）
    3. 由于多层树是一维一维构造出来的，故不支持NN算法

    Range Tree

    1. 在Multi-level Tree的基础上，Range Tree可以将最后一维构建为向量/链表，在定位到倒数第二维的范围后，当前子区间沿之前建立的链接的方式可以在时间内直接定位最终的命中集合的范围

    Interval Tree

    1. Interval Tree用于查询对于任意一个点，有多少个区间包含之
    2. 构建时取区间端点（包括左端点和右端点）的中位数端点为切分点，将被穿过的区间分别按左右端点的顺序进行两次排序，将排序结果储存起来，剩余区间分别按在切分点左右递归进行切分，直到剩余区间数量为0，此时所有区间构建成了一棵树，每个节点可能有多个区间
    3. 由构建过程可知预处理空间复杂度为，时间复杂度
    4. 查询时按在切分点的左中右进行递归，每层由于之前的预处理排序，均可以依次输出各个命中区间的相关信息，时间复杂度为，若只考虑计数，则时间复杂度为

    Splay Tree

    1. 伸展树搜索时，通过旋转把目标结点splay到树根过程中两层两层考虑，若最终还剩单层，则进行一次单旋
    2. 伸展树插入时，将命中节点splay到树根，然后在根部拆开树，将新节点作为根节点进行插入
    3. 伸展树删除时，将命中节点splay到树根，删除树根拆成两颗树，然后将原来的直接后继splay到右侧树的树根（通过在右侧树查找刚刚删除的节点实现），最后拼接
    4. 伸展树时间复杂度采用势能分析确定，势能分析的大概思路是，先给出合适的由状态确定的势能定义，可知基本操作会造成势能变化（一般是基本操作数量越多，势能越下降），再通过对每个执行单元的基本操作数和势能变化的和的分析，得到基本操作总数和势能前后总变化量的和的关系，再由势能的变化规律，最终得到基本操作总数的数量级，也就得到了均摊时间复杂度
    5. 伸展树的势能为所有子树的大小的对数值的总和，取值范围为，再通过缩放给出3种旋转情形的操作次数和势能改变量的和的上界，对这一求和进行累加（其中单旋在每次splay种至多发生1次需要被考虑），可得其取值范围为，从而得到分摊复杂度（见习题解析8-2）
    6. 伸展树的优势是无需记录用于维护平衡性的多余信息，实现简单，分摊复杂度为，局部性强
    7. 伸展树的劣势是单次操作有可能退化到，不适合效率敏感的场合

    Hash

    1. 开放散列对应封闭定址，即插入位置与Hash表状态无关并根据Hash算法唯一确定，链地址为一种典型实现方法，封闭散列对应开放定址，插入位置由Hash算法和试探方法有关，双向平方试探为一种典型实现方法
    2. 哈希表删除时需要进行懒惰删除（查找时视为非空，插入时视为已空），否则会造成试探链断裂
    3. 基数排序的具体算法为，按某一进制编码，从最低位开始，对于每一位，从低位已排序的哈希表中，依次取出元素，并按当前位的值放入新的哈希表中
    4. 基数排序遍历前i位后前i位是有序的
    5. 基数排序适用于确定范围内的整数排序，对于满足常对数密度的整数集，以元素数量作为基数排序可以达到的时间复杂度
    6. 计数排序通过遍历计数，得到各个桶排序后的秩的范围，然后依次遍历待排序序列，根据当前元素所属的桶的起始/结束位置和该桶内已经遍历过的元素数量，来确定当前元素应处的位置
    7. 跳表是一种基于随机过程的数据结构，每一个节点在下一层出现时，在上一层仍出现的概率为等比数列，由此可以保证空间复杂度为，查找、插入、删除的时间复杂度为

    B-Tree

    1. B-Tree的阶数为节点的最大分支个数，对于m阶B-Tree，非根内部节点分支数范围为，根节点分支数范围为，最底层为为外部节点，外部节点的深度均相同（进行手动推演时可利用这点检查），且树高的计算包括外部节点
    2. B-树的高度取值范围为（通过考虑极端情况可以证明）
    3. 红黑树本质就是4阶B-树的另一种形式（“中间”关键码为黑色，“两侧”关键码为红色）
    4. 插入操作一定从叶节点开始，可能导致上溢
    5. 处理上溢需要分裂当前节点，并将推至上层节点相应位置，故上溢可能连续发生，若根节点上溢，则创造出新的根节点，此时树高增加
    6. 删除一定最先发生在叶节点，这需要将位于非叶节点中的待删除关键码与直接后继交换（由B树外部节点深度均相同的性质，后继一定存在且位于叶节点），之后的删除过程中可能出现下溢
    7. 非根节点发生下溢时，先试图通过旋转调整，即在包含至少个关键码的左/右兄弟节点，父节点和自身三者之间进行关键码的移动（同时伴随一条分支的移动，该分支的“挂靠”节点也会发生变化）
    8. 若不存在合适的左右兄弟，则需要从父节点下拉和左/右兄弟分界处的关键码，然后和左/右兄弟节点三者进行合并（同时需要调整父节点的分支），这种情况下下溢可能连续发生，若需要下拉根节点的关键码且根节点只有一个关键码，此时树高下降

    RB Tree

    1. 红黑树添加外部节点（全为黑）便于算法的实现
    2. 红黑树需满足4条规则，即根节点为黑色，外部节点均为黑色，红节点只能有黑孩子，所有外部节点的黑深度相等，这样可以保证红黑树的高度是
    3. 由于红黑树的性质，黑节点一定有兄弟节点

    5. 红黑树删除操作要求（若删除节点非叶节点）先与直接后继交换，只有实际删除节点与其两个子节点（至少有一个为外部节点）都为黑时，才可能需要（排除删除的是根节点或删除后为空树的平凡情况）进行双黑（双黑一词代表当前节点黑高度相比兄弟节点少1）修正（否则无需修正或需要进行简单的颜色转换），分兄弟为黑且有红孩子，兄弟及其两个孩子为黑且父为红，兄弟及其两个孩子为黑且父为黑，兄弟为红这四种情况讨论，其中前两种情况调整立即完成，第三种情况只染色不旋转，第四种情况既旋转又染色，但会随即转化为前两种情况
